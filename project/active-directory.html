Enterprise Active Directory Lab

STACK: Windows Server 2022, Windows 10/11, Sysmon, Splunk/Wazuh

1. The Objective

To simulate a corporate environment, I built a Windows Domain from scratch. The goal was not just connectivity, but security hardening. I wanted to understand how attackers move laterally and how to detect them using event logs.

Figure 1: Domain Controller & Endpoint Architecture


2. Hardening with Group Policy (GPO)

I deployed a Domain Controller and joined Windows 10/11 workstations to the domain. Key security policies implemented included:

  • Disabling generic accounts and enforcing password complexity.
  • Restricting PowerShell execution to Administrators only.
  • Enabling Audit Process Tracking (Event ID 4688) for command-line logging.

3. Detection & Monitoring

I installed Sysmon on all endpoints to capture detailed telemetry. These logs were forwarded to my SIEM (Wazuh/Splunk), allowing me to visualize failed login attempts and flag suspicious process creation in real-time.



Outcome: Successfully detected a simulated "Brute Force" attack and isolated the compromised host within 5 minutes.
